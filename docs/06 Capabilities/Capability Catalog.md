# Capability Catalog

## Project Groundwork

Capabilities are reusable tools that can be enabled inside Workspaces.

They are the building blocks of Groundwork.

A Capability should be designed once and reused across many Workspace Types.

---

# Initial Capabilities

## Mission Control

The main operational view for a Workspace.

Shows current state, priorities, progress, and next actions.

## Tasks

Tracks work that needs to be completed.

Supports owners, due dates, status, and relationship to a Workspace.

## Timeline

Tracks phases, milestones, dates, and deadlines.

Shows how work moves from start to finish.

## People

Manages people connected to a Workspace.

Supports roles, teams, and responsibility.

## Documents

Stores files, links, and structured records connected to a Workspace or Object.

## Communications

Preserves important updates, announcements, and notes in context.

## Budget

Tracks planned and actual financial information connected to a Workspace.

## Inventory

Connects physical resources to a Workspace.

Supports assigned items, modules, readiness, and usage history.

## Forms

Collects structured information.

Supports repeatable intake, feedback, and preparation workflows.

## Registration

Manages attendance, reservations, capacity, and participant records.

## Knowledge Base

Preserves lessons learned, guides, and reusable information.

## Insights

Summarizes progress, patterns, and attention areas.

## Templates

Turns repeated work into reusable structures.

---

# MVP Capabilities

The first MVP should focus on:

1. Mission Control
2. Tasks
3. Timeline
4. People
5. Documents
6. Notes
7. Lessons Learned
8. Templates

---

# Design Standard

A Capability is successful when it helps a Workspace move forward.

If a Capability creates more confusion than clarity, it should be simplified before it is expanded.
