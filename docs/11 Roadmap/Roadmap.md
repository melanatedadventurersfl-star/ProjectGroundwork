# Roadmap

## Project Groundwork

This roadmap defines the early development path for Groundwork.

The roadmap should remain flexible. It is a planning tool, not a cage.

---

# Phase 0: Foundation

Goal:

Define the product before building it.

Deliverables:

- README
- Product Manifesto
- Product Constitution
- Vocabulary
- Core Architecture
- Workspace Ontology
- Experience Ontology
- Capability Catalog
- User Roles
- User Stories
- MVP Definition
- Architecture Decision Records

Status:

In progress.

---

# Phase 1: Product Blueprint

Goal:

Translate the foundation into buildable product specifications.

Deliverables:

- Information Architecture
- Navigation Model
- Workspace Type Definitions
- Capability Specifications
- Object Relationship Map
- Permission Model
- MVP User Flows
- Wireframe Plan

---

# Phase 2: Prototype

Goal:

Create a clickable version of the MVP experience.

Deliverables:

- Home screen wireframe
- Workspace list wireframe
- Create Workspace flow
- Mission Control wireframe
- Task flow
- Timeline flow
- People flow
- Documents flow
- Lessons Learned flow
- Template creation flow

Success Criteria:

A user should be able to walk through planning a real Workspace conceptually, even before full development begins.

---

# Phase 3: Technical Foundation

Goal:

Design the technical system that will support the MVP.

Deliverables:

- Tech stack decision
- Database model
- Authentication plan
- Authorization plan
- API design
- File storage plan
- Deployment plan
- Development workflow

---

# Phase 4: MVP Build

Goal:

Build the first usable version of Groundwork.

MVP Features:

- Organization setup
- Workspace creation
- Workspace overview
- Mission Control
- Tasks
- Timeline
- People
- Documents
- Notes
- Lessons Learned
- Templates

---

# Phase 5: Internal Test

Goal:

Use Groundwork internally on a real Workspace.

Recommended test case:

**Little Camp of Horrors**

Evaluation questions:

- Did Groundwork reduce scattered planning?
- Did Mission Control clarify next actions?
- Did tasks and timelines stay connected?
- Did the Workspace preserve useful knowledge?
- Did the template concept work?

---

# Phase 6: Private Beta

Goal:

Invite a small number of trusted users or organizations to test Groundwork.

Focus:

- Usability
- Workspace clarity
- Task flow
- Documentation gaps
- Template usefulness
- Role clarity

---

# Phase 7: Expanded Capabilities

Goal:

Add capabilities after the Workspace model is proven.

Possible expansions:

- Registration
- Forms
- Inventory depth
- Budget depth
- Insights
- AI Assistance
- Public pages
- Client booking flows
- Mobile app

---

# Roadmap Principle

Do not expand Groundwork because an idea is interesting.

Expand when the core Workspace model reveals a real operational need.

Strong foundations first.

Feature fireworks later.
