# User Stories

## Project Groundwork

This document captures the initial user stories for Groundwork.

User stories describe what people need to accomplish and why it matters.

They are not final requirements. They are the starting point for product design, backlog planning, and MVP scoping.

---

# Story Format

Groundwork user stories use this format:

> As a [role], I want to [action], so that [outcome].

Each story should connect to a real operational need.

---

# Organization Owner Stories

- As an Organization Owner, I want to create an Organization, so that I can manage people, work, settings, and branding in one place.
- As an Organization Owner, I want to assign roles, so that each person has the right responsibility.
- As an Organization Owner, I want to view active Workspaces, so that I understand what needs attention.

---

# Workspace Lead Stories

- As a Workspace Lead, I want to create a Workspace, so that a body of work has a dedicated home.
- As a Workspace Lead, I want to configure capabilities, so that the Workspace fits the work being done.
- As a Workspace Lead, I want to view Mission Control, so that I understand status, priorities, and next actions.
- As a Workspace Lead, I want to assign tasks, so that everyone knows what they own.
- As a Workspace Lead, I want to archive completed work, so that knowledge is preserved.
- As a Workspace Lead, I want to create a Template, so that similar work can start faster next time.

---

# Team Lead Stories

- As a Team Lead, I want to view team tasks, so that I can keep my area of work organized.
- As a Team Lead, I want to update team progress, so that Workspace leadership knows what is complete.
- As a Team Lead, I want to view team resources, so that my team knows what it is responsible for.

---

# Worker and Volunteer Stories

- As a Worker or Volunteer, I want to view my assignments, so that I know what I need to do.
- As a Worker or Volunteer, I want to update task status, so that leads know what has been completed.
- As a Worker or Volunteer, I want to view my schedule, so that I know when I am needed.

---

# Member and Guest Stories

- As a Member or Guest, I want to view public Experiences, so that I can decide whether I want to attend.
- As a Member or Guest, I want to register for an Experience, so that my spot is reserved.
- As a Member or Guest, I want to view preparation details, so that I know what to expect.

---

# Product Stories

- As a user, I want to reuse past work, so that I do not have to start from scratch.
- As a user, I want to preserve lessons learned, so that future work improves.
- As a user, I want to search across Workspaces, so that I can find prior knowledge.
- As a user, I want Groundwork to show what comes next, so that I can move forward confidently.

---

# MVP Story Candidates

The strongest MVP stories are:

1. Create a Workspace
2. Configure Workspace capabilities
3. View Mission Control
4. Create and assign Tasks
5. Add People to a Workspace
6. Add Documents to a Workspace
7. Track Timeline milestones
8. Capture Lessons Learned
9. Archive a Workspace
10. Create a Template from a Workspace

These stories should shape the first usable prototype.
