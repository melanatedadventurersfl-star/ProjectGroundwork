# MVP Definition

## Project Groundwork

This document defines the initial Minimum Viable Product for Groundwork.

The MVP should prove the core product idea:

> A Workspace can become the central home for meaningful work.

The MVP does not need to support every future capability.

It needs to help a user create, organize, manage, and preserve one real Workspace.

---

# MVP Goal

Build the smallest useful version of Groundwork that allows an organizer to manage a real body of work from one place.

The MVP should prove that Workspaces reduce scattered planning and make next actions clearer.

---

# MVP User

Primary MVP user:

**Workspace Lead**

This person creates and manages a Workspace.

Secondary users:

- Team Lead
- Worker or Volunteer
- Organization Owner

The first version should focus on organizer value before expanding deeply into public member or client experiences.

---

# MVP Scope

## Included

The MVP should include:

- Organization setup
- Workspace creation
- Workspace overview
- Mission Control
- People connected to a Workspace
- Tasks
- Timeline milestones
- Documents
- Basic communications or notes
- Lessons Learned
- Archive Workspace
- Create Template from Workspace

## Not Included Yet

The MVP does not need to include:

- Full payment processing
- Full ticketing platform
- Advanced analytics
- Complex automations
- Native mobile apps
- Multi-chapter enterprise controls
- Advanced inventory scanning
- Public marketplace

These may come later after the core Workspace model is proven.

# MVP Capabilities

The MVP should focus on these capabilities:

## Mission Control

A summary of the Workspace state, priorities, blockers, and next actions.

## Tasks

Create, assign, and complete tasks.

## Timeline

Track milestones and important dates.

## People

Add people and assign simple roles.

## Documents

Attach files and links to the Workspace.

## Notes

Capture internal notes and useful context.

## Lessons Learned

Capture what should be remembered for future Workspaces.

## Templates

Turn a Workspace into a reusable starting point.

---

# MVP Success Criteria

The MVP succeeds if a user can:

- Create a Workspace
- Understand what the Workspace is for
- Add people
- Add tasks
- Track progress
- Store supporting documents
- Capture useful notes
- Preserve lessons learned
- Reuse the Workspace as a Template

The MVP should make the user feel more prepared and less scattered.

---

# First Real Test Case

The first test case should be a real operational use case from Melanated Adventurers or Build-A-Camp.

Recommended test case:

**Little Camp of Horrors**

This test case is useful because it includes:

- Planning
- Venue details
- Crew
- Tasks
- Budget
- Inventory
- Communications
- Documents
- Lessons Learned

If Groundwork can help manage this Workspace clearly, the core model is promising.

---

# MVP Principle

The MVP should not try to prove that Groundwork can do everything.

It should prove that the Workspace model makes real work easier to manage.
