# User Flows

## Project Groundwork

This document defines the initial product flows for Groundwork.

User flows describe how people move from intention to completion.

---

# Purpose

User flows help connect product strategy to screen design and implementation.

Each flow should define:

- User goal
- Starting point
- Steps
- Success state
- Related screens
- MVP notes

---

# Flow 1: Create a Workspace

## User Goal

Create a dedicated home for a new body of work.

## Primary User

Workspace Lead

## Starting Point

Home or Workspace List

## Steps

1. User selects `Create Workspace`.
2. User chooses Workspace Type.
3. User enters name and description.
4. User assigns owner.
5. User selects start date and target date.
6. User chooses template or starts blank.
7. User confirms enabled capabilities.
8. Groundwork creates the Workspace.
9. User lands on Mission Control.

## Success State

The Workspace exists and is ready for setup.

## Related Screens

- Home
- Workspace List
- Create Workspace
- Mission Control

---

# Flow 2: Set Up a New Workspace

## User Goal

Turn an empty Workspace into an actionable planning space.

## Primary User

Workspace Lead

## Starting Point

Mission Control

## Steps

1. User opens a new Workspace.
2. Mission Control shows setup guidance.
3. User adds timeline milestones.
4. User adds initial tasks.
5. User adds people and roles.
6. User adds key documents or links.
7. User reviews Mission Health.
8. Groundwork updates suggested next actions.

## Success State

The Workspace has enough structure to begin real work.

---

# Flow 3: Manage Tasks

## User Goal

Create, assign, and track work.

## Primary Users

- Workspace Lead
- Team Lead
- Worker
- Volunteer

## Starting Point

Mission Control, My Work, or Workspace Tasks

## Steps

1. User opens Tasks.
2. User creates or selects a task.
3. User assigns owner.
4. User sets due date.
5. User connects the task to a milestone if needed.
6. User updates status as work progresses.
7. Groundwork reflects task status in Mission Control.

## Success State

Work is assigned, visible, and trackable.

---

# Flow 4: Add People to a Workspace

## User Goal

Connect people to the work and clarify responsibility.

## Primary User

Workspace Lead

## Starting Point

Mission Control or Workspace People

## Steps

1. User opens People.
2. User selects existing Person or creates a new one.
3. User assigns role.
4. User optionally assigns Team.
5. User saves.
6. Groundwork updates People summary and related responsibilities.

## Success State

The right people are connected to the Workspace with clear roles.

---

# Flow 5: Add a Document

## User Goal

Store important supporting material in context.

## Primary Users

- Workspace Lead
- Team Lead
- Organization Owner

## Starting Point

Mission Control or Workspace Documents

## Steps

1. User opens Documents.
2. User adds a file or link.
3. User enters title and type.
4. User optionally connects it to a task, milestone, person, venue, or vendor.
5. User saves.
6. Document appears in Workspace Documents and relevant summaries.

## Success State

The document is preserved and findable in the right context.

---

# Flow 6: Capture a Lesson Learned

## User Goal

Preserve useful knowledge from completed work.

## Primary User

Workspace Lead

## Starting Point

Mission Control or Lessons Learned

## Steps

1. User opens Lessons Learned.
2. User adds a lesson.
3. User categorizes the lesson.
4. User connects it to related objects if helpful.
5. User saves.
6. Lesson becomes available for future templates and reference.

## Success State

Knowledge is preserved instead of disappearing after the work is done.

---

# Flow 7: Create Template from Workspace

## User Goal

Reuse a successful Workspace structure.

## Primary User

Workspace Lead or Organization Owner

## Starting Point

Mission Control, Workspace Settings, or Templates

## Steps

1. User selects `Create Template`.
2. Groundwork shows what can be included.
3. User chooses tasks, timeline, roles, documents, notes, and lessons.
4. User names the template.
5. User saves the template.
6. Template becomes available for future Workspaces.

## Success State

Repeat work can start faster next time.

---

# Flow 8: Archive Workspace

## User Goal

Close active work while preserving its history.

## Primary User

Workspace Lead

## Starting Point

Mission Control or Workspace Settings

## Steps

1. User selects `Archive Workspace`.
2. Groundwork prompts user to review open tasks and lessons learned.
3. User confirms archive.
4. Workspace status changes to Archived.
5. Workspace remains searchable and reusable.

## Success State

Completed work is preserved without cluttering active views.

---

# MVP Flow Priority

The MVP should prioritize:

1. Create Workspace
2. Set Up Workspace
3. Manage Tasks
4. Add People
5. Add Documents
6. Capture Lessons Learned
7. Create Template
8. Archive Workspace

---

# Flow Standard

A Groundwork flow succeeds when the user reaches a useful outcome with minimal uncertainty.

Each flow should reduce confusion, preserve context, and make the next step clear.
